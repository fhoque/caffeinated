Home Page
    Description:
    The "Get Me Caffeinated!" homepage is designed for simplicity above all else. Firstly, we hope to present the two main functions of the site up front: Learning about coffee and finding new coffee recipes. The available options are presented as large buttons to avoid any confusion on the part of the user. As was described in our interface goals, our goal was to offer something for both new users and users with alot of domain knowledge. This page allows the user to specify their skill level when searching for recipes, so that the rest of their interactions with the program will meet their needs. We also decided to include a logo in the top-left corner of each page which also acts as a link to the home page. This will allow users to always know what site they are on, as per the advice in "Don't make me think!"
    Fundamentals
      Description:
      The "Fundamentals" screen is designed to clearly present users with choices that relate to learning about coffee. The "breadcrumbs" at the top-left of the screen help users to tell where they are in the site. While some of the options presented do take the user to another page, others simply trigger partial page updates in order to show the new content. This is done in order to reduce navigation and therefore lessen excise. In fact, one of our main design goals was to limit excise tasks for the user, and in particular to minimize navigation. With that in mind, we tried to always give the user ways to access the rest of the site content from each screen, so that they won't have to navigate to another page unless that page is doing something useful for them.
      Terminology
        Description:
        An accordion UI widget will be used for the list of coffee terminology. If the user selects a specific term, the entry for that term will expand and reveal a definition along with related information. By using this widget, it is our hope that we can reduce excessive navigation and scrolling in particular. The terms are presented in alphabetic order, because we believe this will best fit the user's mental model (like a dictionary or glossary).
        Terminolgy - Caffeine
          Description:
          When a term is selected, the accordion widget expands the corresponding entry to reveal the definition.
          Test your knowledge
            Description:
            The test your coffee knowledge is meant to test the user on terminology understanding. It focuses more on textual descriptions of various terms as opposed to the more visual matching game. The user will gain confidence with coffee terminology and eventually be able to go through the beginner or advanced drink-finder pages with a better understanding. The drink finder option is also obscured until the other options are used so that the beginner user will not be confused by the recipes that they receive.
            Matching Game
              Description:
              The matching game contains a variety of visual trials and grouping activities that tests the user's understanding of terms, tools, and miscellaneous coffee paraphernalia. Some activities include visual multiple choice, logical grouping of items, and ordering based on a drink's properties (hot/cold, weak/strong, etc), and which of these does not belong. The user will mostly be clicking and in some cases clicking and dragging pictures. The goals is to link knowledge learned in terminology and the text quizzes with visuals.
              Beginners
                Description:
                The beginner's questionaire is designed to provide beginner coffee drinkers with very general questions that the site will use to guide them to coffee drinks that they might like. The ranking at the bottom will allow users to drag-and-drop items in order to create an ordered list of their coffee drinking preferences.

                Note: There will probably be more questions in the finished form (they are still being determined), but all of them will follow the same general format.
                Experts
                  The purpose of the advanced questionaire is to allow user's with alot of coffee related knowledge to find new coffee drinks. The page provides sliders which allow the user to give their exact preferences with regard to a number of coffee related criteria. As with the beginner's questionaire, the application will find the drink that most matches the user's preferences. This particular questionaire is spread across two pages in order to allow for more criteria. One of our primary personas is that of someone who already knows how to make coffee, but would like to find recipes that better match their tastes. This page attempts to facilitate that goal by allowing the user to specify their tastes with greater specificity.
                  Experts Pt.2
                    Description:
                    This screen simply provides more criteria for the user to customize their drink settings. If there are more criteria added later, they will follow this layout.
                    Results
                      Description:
                      The results page lists the most applicable beverages for the user based on his or her form data. The user is given a brief overview of the beverage and then must select one to proceed and receive a detailed recipe on how to make this drink. If they do not like any of the beverages, they can try a new search.
                      Directions
                        Description:
                        After a user has provided their drink preferences in a questionaire, they are directed to this screen, which tells them which coffee drink best fits their tastes. The screen also provides a list of both the ingredients and equipment they will need in order to prepare the drink. For each entry in both lists, a link is provided which allows the user to find out where they can acquire the item if they do not already have it. Finally, a button allows users to quickly view an interactive slide show which describes the steps necessary to prepare the drink.
                        Where can I find this
                          Description:
                          When the user selects the "Where can I find this?" option, a pop-up window will appear which will show the user a list of stores that sell the chosen ingredient. This window is not modal, so it can be moved or minimized as the user sees fit. In general we have saught to minimize any form of modeless feedback in the site in order not to disrupt the user's flow.
                          Preparation
                            The slideshow provides users with step-by-step directions for preparing their chosen drink. The user can manually navigate to the next or previous slides, or they can simply play the slideshow, in which case the slides will be shown automatically with a set time duration between each slide. A link is also provided that will automatically cause the browser to scroll to the top of the page so that the user can view information about ingredients or equipment. The goal here is to minimize scrolling while also keeping all of the information on the same page. Conceivably, a user with a large enough screen could also view everything at once, so this will fit theire needs very well.

                            We had originally planned to have the slideshow on a seperate page, but we decided it would be inconvenient for the user to have to navigate back to the previous page if they needed to view information about ingredients or equipment.
                            Print
                              Description
                              This opens as a new page to automatically print the ingredients, equipment, and preparation lists without navigation, weblinks, or iteration (that would only make sense on the web, not in print). The automation removes a few excise tasks. The browser title clearly states which drink it is displaying directions for.
